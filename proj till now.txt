The Render deployment is failing with:

ModuleNotFoundError: No module named 'backend'

Reason:
Render is using:

* Root Directory = backend

So Python is already running INSIDE the backend folder.

However, the code still contains imports like:

from backend.routers import ...
from backend.services import ...
from backend.agents import ...

These imports work locally from the project root but fail on Render because there is no parent `backend` package in the runtime path.

Please do ALL of the following carefully:

1. Search the entire `backend/` directory for imports starting with:
   `from backend.`
   and
   `import backend.`

2. Convert them to local package imports.

Examples:

Change:

```python
from backend.routers import research, memory, health
```

To:

```python
from routers import research, memory, health
```

Change:

```python
from backend.services.memory import MemoryService
```

To:

```python
from services.memory import MemoryService
```

3. Ensure all imports remain valid and consistent across:

* agents/
* routers/
* services/
* utils/
* models/
* main.py

4. Verify there are no remaining `backend.` import prefixes anywhere in the backend codebase.

5. Ensure the project structure remains compatible with:

* Render deployment

6. After modifications:

* run a quick import validation check if possible
